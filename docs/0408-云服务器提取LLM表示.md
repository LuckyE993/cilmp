# `cilmp/scripts/generate_llm_representations.py` 部署说明

本文只解决一件事：在一台新机器上，单独运行 `cilmp/scripts/generate_llm_representations.py` 生成 CILMP 所需的 `llm_representations/*.pth` 特征文件。

不包含训练环境，不要求安装 `src/dassl`，也不要求准备数据集图片。这个脚本只会根据类别名向 LLM 提问，然后保存每个类别对应的隐藏层表征。

## 1. 先确认这几个前提

### 1.1 推荐硬件

- 推荐 Linux + NVIDIA GPU。
- 如果你用的是 8B 级别模型，例如 `Meta-Llama-3-8B-Instruct`，更稳妥的是 `24GB+` 显存。
- `16GB` 显存大概率会在加载模型或 `output_hidden_states=True` 这一步 OOM。
- 纯 CPU 也能跑，但会很慢，而且建议至少有较大的系统内存。

### 1.2 磁盘空间

- 预留至少 `30GB~50GB` 磁盘空间给模型和缓存。

### 1.3 模型维度约束

当前仓库的 CILMP 训练代码把 LLM 特征维度写死为 `4096`，所以你要优先选择 `hidden_size=4096` 的 decoder-only 模型。

例如：

- `meta-llama/Meta-Llama-3-8B-Instruct`
- `meta-llama/Llama-2-7b-chat-hf`
- `mistralai/Mistral-7B-Instruct-v0.3`

如果你生成的是非 `4096` 维特征，当前仓库训练代码不能直接用。

## 2. 拿到项目并进入根目录

如果你已经把项目拷到新机器了，直接进入仓库根目录即可：

```bash
cd /path/to/cilmp
```

如果你要重新拉代码：

```bash
git clone <your-repo-url> cilmp
cd cilmp
```

下面所有命令都默认在仓库根目录执行，也就是这个目录下能看到：

- `README.md`
- `deploy.md`
- `cilmp/scripts/generate_llm_representations.py`

## 3. 创建最小 Conda 环境

这里按“只跑这个脚本”的最小环境来配，不走整仓训练依赖。

### 3.1 创建环境

```bash
conda create -n cilmp-llm python=3.10 -y
conda activate cilmp-llm
python --version
```

### 3.2 安装 PyTorch

如果新机器有 NVIDIA GPU，推荐安装 CUDA 12.1 版本的 PyTorch：

```bash
conda install -y pytorch torchvision torchaudio pytorch-cuda=12.1 -c pytorch -c nvidia
```

如果新机器没有 GPU，装 CPU 版：

```bash
conda install -y pytorch torchvision torchaudio cpuonly -c pytorch
```

### 3.3 安装脚本依赖

```bash
pip install -U pip
pip install "transformers>=4.41" "accelerate>=0.30" safetensors sentencepiece tiktoken "huggingface_hub[cli]"
pip install modelscope
```

说明：

- `modelscope` 只有在你准备用 `--model-source modelscope` 时才需要，但直接装上最省事。
- `accelerate` 是给 `from_pretrained(..., low_cpu_mem_usage=True)` 这条加载路径兜底。
- 这里不需要安装 `cilmp/requirements.txt`，因为那个是整仓训练环境，不是这个脚本的最小依赖。

## 4. 验证环境是否正常

```bash
python -c "import torch, transformers; print('torch=', torch.__version__); print('cuda_available=', torch.cuda.is_available()); print('transformers=', transformers.__version__)"
python cilmp/scripts/generate_llm_representations.py --help
```

如果上面第二条能正常打印帮助信息，说明脚本依赖已经齐了。

如果你是 GPU 机器，也建议先看一下：

```bash
nvidia-smi
```

## 5. 运行前先决定模型来源

这个脚本支持三种模型来源：

1. Hugging Face 在线下载
2. ModelScope 在线下载
3. 本地模型目录

如果你的机器能稳定访问 Hugging Face，就直接走 Hugging Face。

如果机器在国内网络环境，更建议用 ModelScope 或本地模型目录。

## 6. 运行方式一：Hugging Face

### 6.1 如果模型是 gated repo，先登录

像 `meta-llama/Meta-Llama-3-8B-Instruct` 这类模型，通常需要先在 Hugging Face 页面同意协议，然后登录：

```bash
huggingface-cli login
```

### 6.2 执行命令

下面以 `derm7pt` 为例：

```bash
python cilmp/scripts/generate_llm_representations.py \
  --dataset derm7pt \
  --model-source huggingface \
  --model-name meta-llama/Meta-Llama-3-8B-Instruct \
  --trust-remote-code \
  --save-metadata
```

如果你想指定 Hugging Face 缓存目录：

```bash
python cilmp/scripts/generate_llm_representations.py \
  --dataset derm7pt \
  --model-source huggingface \
  --model-name meta-llama/Meta-Llama-3-8B-Instruct \
  --cache-dir /data/hf-cache \
  --trust-remote-code \
  --save-metadata
```

## 7. 运行方式二：ModelScope

如果你更适合从 ModelScope 下载：

```bash
python cilmp/scripts/generate_llm_representations.py \
  --dataset derm7pt \
  --model-source modelscope \
  --model-name LLM-Research/Meta-Llama-3-8B-Instruct \
  --cache-dir /data/modelscope-cache \
  --trust-remote-code \
  --save-metadata
```

如果模型已经缓存过了，只想离线读取：

```bash
python cilmp/scripts/generate_llm_representations.py \
  --dataset derm7pt \
  --model-source modelscope \
  --model-name LLM-Research/Meta-Llama-3-8B-Instruct \
  --local-files-only \
  --trust-remote-code \
  --save-metadata
```

如果你想把 ModelScope 快照明确落到一个指定目录：

```bash
python cilmp/scripts/generate_llm_representations.py \
  --dataset derm7pt \
  --model-source modelscope \
  --model-name LLM-Research/Meta-Llama-3-8B \
  --modelscope-local-dir ../models/Meta-Llama-3-8B \
  --trust-remote-code \
  --save-metadata
```

## 8. 运行方式三：本地模型目录

如果模型文件已经提前下载好了，最稳的是直接传本地路径：

```bash
python cilmp/scripts/generate_llm_representations.py \
  --dataset derm7pt \
  --model-source local \
  --model-name /data/models/Meta-Llama-3-8B-Instruct \
  --trust-remote-code \
  --save-metadata
```

这时脚本不会去远端拉模型。

## 9. 常用参数说明

### 9.1 内置数据集名

脚本内置支持这些 `--dataset`：

- `adam`
- `aptos2019`
- `bloodmnist`
- `busi`
- `chaoyang`
- `cpn_x_ray`
- `derm7pt`
- `dermamnist`
- `fetal_us`
- `isic`
- `kvasir`
- `odir`
- `odir2`
- `pneumonia2`
- `pneumonia3`

### 9.2 自定义类别

如果你不是跑内置数据集，而是自己指定类别：

```bash
python cilmp/scripts/generate_llm_representations.py \
  --classes "normal" "benign" "malignant" \
  --output-subdir my_dataset \
  --model-source local \
  --model-name /data/models/Meta-Llama-3-8B-Instruct \
  --trust-remote-code \
  --save-metadata
```

生成结果会落到：

```bash
cilmp/llm_representations/my_dataset/
```

### 9.3 常用可选项

- `--overwrite`：如果同名 `.pth` 已存在，就覆盖。
- `--save-metadata`：额外保存 `metadata.json`，记录每个类别的提问和生成结果。
- `--chat-template never`：如果你用的是 base 模型而不是 instruct/chat 模型，建议加上。
- `--max-new-tokens 64`：如果显存紧张，可以把默认的 `128` 再压低一点。
- `--device cpu`：强制走 CPU。
- `--dtype float32`：CPU 模式下建议显式指定。

例如，CPU 模式：

```bash
python cilmp/scripts/generate_llm_representations.py \
  --dataset derm7pt \
  --model-source local \
  --model-name /data/models/Meta-Llama-3-8B-Instruct \
  --device cpu \
  --dtype float32 \
  --trust-remote-code \
  --save-metadata
```
python cilmp/scripts/generate_llm_representations.py \
  --dataset derm7pt \
  --model-source local \
  --model-name ../Meta-Llama-3-8B \
  --device cpu \
  --dtype float32 \
  --trust-remote-code \
  --save-metadata
## 10. 输出位置和校验方法

默认输出目录是：

```bash
cilmp/llm_representations/<dataset>/
```

以 `derm7pt` 为例，结果会在：

```bash
cilmp/llm_representations/derm7pt/
```

如果加了 `--save-metadata`，该目录下还会有一个 `metadata.json`。

### 10.1 查看生成了哪些文件

```bash
find cilmp/llm_representations/derm7pt -maxdepth 1 -type f | sort
```

### 10.2 检查某个 `.pth` 的形状

```bash
python -c "import torch; x=torch.load('cilmp/llm_representations/derm7pt/carcinoma.pth', map_location='cpu'); print(tuple(x.shape), x.dtype)"
```

期望看到的形状类似：

```bash
(32, 4096)
```

第一维是 decoder layer 数，第二维必须是 `4096`。

## 11. 常见报错和处理

### 11.1 `ValueError: Model hidden_size=..., but expected 4096`

原因：

- 你选的模型隐藏维度不是 `4096`。

处理：

- 换成 `hidden_size=4096` 的模型。
- 如果你的目标是给当前这个 CILMP 仓库训练用，不要简单改 `--expected-hidden-size` 来绕过检查，因为训练代码本身也写死了 `4096`。

### 11.2 `CUDA out of memory`

原因：

- 8B 模型占用太大。

处理：

- 优先换更大显存的 GPU。
- 把 `--max-new-tokens` 降低，例如 `64`。
- 改成 `--device cpu`，但会很慢。
- 当前脚本没有做量化加载，也没有做 `device_map=auto` 分片，所以不要默认指望 16GB 卡能稳跑 8B 模型。

### 11.3 Hugging Face 403 / gated repo / 权限错误

原因：

- 没有在 Hugging Face 页面同意模型协议，或者没登录。

处理：

- 去模型页面先同意协议。
- 重新执行 `huggingface-cli login`。
- 或者改走 `--model-source modelscope` / `--model-source local`。

### 11.4 `ModuleNotFoundError: No module named 'modelscope'`

原因：

- 你用了 `--model-source modelscope`，但没有安装 `modelscope`。

处理：

```bash
pip install modelscope
```

### 11.5 已有文件但脚本没有重新生成

原因：

- 脚本默认遇到已存在的 `.pth` 会跳过。

处理：

```bash
python cilmp/scripts/generate_llm_representations.py \
  --dataset derm7pt \
  --model-source local \
  --model-name /data/models/Meta-Llama-3-8B-Instruct \
  --trust-remote-code \
  --overwrite \
  --save-metadata
```

### 11.6 chat 模板不合适，生成文本异常

处理：

- instruct/chat 模型：保持默认 `--chat-template auto`
- base 模型：尝试加 `--chat-template never`

## 12. 一条最稳的推荐命令

如果你已经把模型提前放到本地目录，这是最稳的一条：

```bash
python cilmp/scripts/generate_llm_representations.py \
  --dataset derm7pt \
  --model-source local \
  --model-name /data/models/Meta-Llama-3-8B-Instruct \
  --trust-remote-code \
  --save-metadata
```

原因很简单：

- 不依赖外网
- 不受 Hugging Face 权限影响
- 不受 ModelScope 临时下载状态影响

## 13. 结论

要在新机器上单独跑这个脚本，最小流程就是：

1. 创建 `conda` 环境。
2. 安装 `PyTorch + transformers + accelerate + safetensors`，需要的话再装 `modelscope`。
3. 准备一个 `hidden_size=4096` 的 decoder-only 模型。
4. 在仓库根目录执行 `python cilmp/scripts/generate_llm_representations.py ...`。
5. 检查 `cilmp/llm_representations/<dataset>/` 下生成的 `.pth` 文件。
